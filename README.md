### # 開発スタイル / 価値観

#### 設計の根本的な揺り戻しを恐れない

- 「過去にこう実装したから」より「今の判断軸ならどうか」で書き直すスタイル。判断軸が変われば過去のコードもゼロベースで作り直す。

#### Claude Code の使い方

- 業務でも個人開発でも Claude Code をヘビー利用している。独自スキル / コマンドを自作して運用に組み込む派。
- 作業ログを daily ファイルに残し、セッションをまたぐ重要情報は memory に格上げする `/memo` コマンドを自作・運用。
- セッション間でコンテキストが消える前提で、記録の積み上げで文脈を維持する方針。
